================================================================================
OUTPUT FORMAT
================================================================================

JSON structure with drift events containing:
- repository: Repository name (owner/repo)
- commit_sha: Unique commit identifier
- commit_message: Full commit message
- commit_date: Timestamp of fix
- author: Commit author
- file: Modified file path
- patch: Git diff of changes
- before_segments: Code-doc pairs BEFORE fix (Drifted)
- after_segments: Code-doc pairs AFTER fix (Consistent)

Each segment includes:
- filename: File name
- start_line: Line number
- code: Function/class code with context
- documentation: Associated docstring

================================================================================
USAGE EXAMPLES
================================================================================

Basic usage:
  python3 drift_miner.py

Custom repositories:
  python3 drift_miner.py --repos scipy/scipy numpy/numpy

Limit search:
  python3 drift_miner.py --max-commits 50

With GitHub token:
  python3 drift_miner.py --token YOUR_TOKEN

Custom output:
  python3 drift_miner.py --output my_analysis.json

Programmatic:
  from drift_miner import DriftMiner
  miner = DriftMiner()
  events = miner.mine_repository('numpy/numpy', max_commits=100)
