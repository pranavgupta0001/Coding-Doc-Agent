Original Document
Task Management API — Product Specification v1.0

This document describes the required capabilities of the Task Management REST API.

Authentication & Authorization
1. Users must authenticate via JWT tokens. Tokens expire after 24 hours.
2. The system must implement RBAC with three roles: admin, editor, and viewer.
3. All API endpoints must require a valid JWT token in the Authorization header.
4. Failed login attempts exceeding 5 tries must lock the account for 15 minutes.

Data & Persistence
5. Task records must be persisted to a relational database (PostgreSQL recommended).
6. All personally identifiable information (PII) must be encrypted at rest using AES-256.
7. The system must retain audit logs for all write operations for 90 days.

Performance
8. API response time must not exceed 200ms at the 99th percentile under normal load.
9. The system must support at least 500 concurrent users without degradation.

Integrations & Real-time
10. The API must expose REST endpoints for CRUD operations on tasks and projects.
11. Real-time task update notifications must be delivered via WebSocket connections.
12. The system must apply rate limiting of 100 requests per minute per user.

Compliance
13. The system must maintain HIPAA-compliant audit logging for all data access events.
14. Users must be able to request deletion of their data (GDPR right to erasure).

Security
15. All data in transit must be protected by TLS 1.2 or higher (HTTPS only).
16. The system must sanitize all user inputs to prevent SQL injection and XSS attacks.
User Prompt
Please review the task management API implementation and verify it correctly implements
the product specification.

The implementation should handle user authentication with JWT, expose REST endpoints
for task management, and support real-time notifications. Confirm that rate limiting,
encryption, and HIPAA-compliant audit logging are all present and functional.
